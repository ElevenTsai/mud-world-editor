# 《万族志MUD》江州篇·地图设计问题分析报告

---

## 一、地图连接性问题（Critical — 直接导致游戏性崩溃）

### 1.1 出口不对称 / 方向矛盾

在文字MUD中，出口方向必须严格双向对称，否则玩家将无法返回或陷入迷失。以下是所有不对称连接：

**江州鱼埠**

| 问题房间 | 错误出口 | 矛盾点 |
|---------|---------|--------|
| 蓄鱼仓 | 东 → 江州鱼市 | 但江州鱼市的**西**出口指向「栖浪酒家」，两者没有对应连接 |
| 修辑坊 | 南 → 栖浪酒家（小巷） | 但栖浪酒家的**西**出口指向修辑坊，南≠西，方向自相矛盾 |
| 捕鱼房 | **缺少南出口** | 江州官仓的北出口指向捕鱼房，玩家从官仓北上后无法返回 |
| 海港商会 | 仅有西出口 | 其余三个方向未定义，形成死胡同 |

**云栖城**

| 问题房间 | 错误出口 | 矛盾点 |
|---------|---------|--------|
| 烟舟坊 | 东 → 青蔬坊（上行栈道） | 但"东"方向描述为"沿崖壁栈道上行"，东/上语义混用 |
| 青蔬坊 | 南 → 云舟修葺台（沿石阶**上行**） | "南"方向但描述为"上行"，方向与海拔相反，极易混淆 |
| 市集 | 南 → 烟舟坊 | 但烟舟坊的南出口也指向市集，两者互相指向对方，无限循环 |
| 望霄楼 | 南 → 市集 | 但市集的**东**出口指向望霄楼，南≠东，方向矛盾 |
| 云舟修葺台 | 上（竹篮）→ 从烟舟坊可升上来 | 但云舟修葺台本身无下/下行出口返回烟舟坊 |

### 1.2 概览连接图与实际房间描述冲突

顶部连接图标注：
```
清鳞分拣厂 → 蓄鱼仓 → 捕鱼房
```
箭头为单向（→），但实际各房间的出口是双向的（↔）。这个图是错误的，误导了设计意图。

**修改建议：** 统一使用 ↔ 表示双向通行，用 → 仅表示单向触发（如传送）。

## 二、任务物品断链问题（Critical — 任务无法完成）

### 2.1 尾橹来源空缺

- 修辑坊主线任务要求：去捕鱼房取回一支「特制的尾橹」
- 但捕鱼房的 NPC 描述（孙老六、水生）及任务中，**完全没有提及尾橹**，也没有说明由谁保管、如何获取

**修改建议：** 在捕鱼房的「渔歌里的故事」任务中，加入孙老六或水生保管旧尾橹的情节，顺带作为支线或小任务奖励交出。示例：
> 孙老六唱完渔歌后，指了指墙角："那根橹，是我年轻时出大潮用的，现在腿跛了，用不着了，你们要拿去就拿去吧。"

### 2.2 云雾茶用途断链

- 附录物品表标注：云雾茶用于「望霄楼支线，换取情报」
- 但望霄楼的全部 NPC（温老先生、柳三变、文屿）和任务描述中，**没有任何关于云雾茶的对话或触发条件**

**修改建议：** 在望霄楼增加一条触发：将云雾茶赠给温老先生，他因感动（云雾茶是他家乡之物）而主动讲述通玄门或水兽的旧事，给出关键线索。

## 三、剧情逻辑问题（Major — 破坏叙事完整性）

### 3.1 失踪的三名船员——悬案永远悬而不决

- 渔户营的主线任务「失踪的渔船」设置了赵船主的船和三名船员失踪的悬念
- 但最终结局明确写道："渔船失踪事件**虽未找到人**"
- 对于MUD玩家来说，接了一个"找人"类任务，结果找不到人，体验感很差，且这三人的命运完全没有交代

**修改建议（二选一）：**

**方案A（宿命线）：** 在玄章镇潮阁一层的石室中增加遗物（一件破渔网、一块船牌），以环境叙事说明三人已遇难于水兽躁动，给玩家一个收尾感，并可触发特殊对话（回到渔户营告知赵船主）。

**方案B（救援线）：** 三人被困在镇潮阁石礁的洞穴中（水兽扰动时被困），玩家「叩钟镇潮」后水面平静，可回到码头驾船将三人救出，完整关闭任务。**推荐此方案**，因为它让主线的解决行动（叩钟）产生直接的、可见的英雄果效，叙事完整性更好。

### 3.2 洪大爷关键信息被锁在"付钱"行为后

- 洪大爷的信息是主线核心：「钟声响起便是有异动」
- 但触发条件是：「若帮他付了酒钱，会透露……」
- 如果玩家钱不够或没有想到主动付酒钱，这条关键主线信息将永远缺失

在MUD游戏中，**主线关键信息不应被可选的付费行为锁住**，否则会导致任务无法推进且没有明确提示。

**修改建议：** 将付钱改为更低门槛的触发条件。例如：
> 洪大爷若看到你携带「镇潮令」或「镇潮录」，会主动开口；或者只需和他多聊几句（2-3轮对话）即可触发，付钱改为增加情感好感度，带来额外内容，而非锁主线。

### 3.3 玄章镇潮阁内部楼层为单向通行

```
阁一层 → 阁二层 → 阁三层（钟楼）
```

在MUD中，单向箭头意味着进入三层后无法下楼。若玩家漏了什么，或者想再次触发残魂对话，完全无法返回。

**修改建议：** 改为双向 ↔，并在三层增加「下行」出口。

## 四、地图结构设计问题（Major）

### 4.1 地理逻辑错乱——蓄鱼仓的位置自相矛盾

根据实际出口推导各房间的地理坐标：

```
[活鱼递运廊] [清鳞分拣厂] [渔户营]
             [蓄鱼仓]   [江州鱼市][海港商会]
[江州官仓]   [捕鱼房]   [栖浪酒家]
             [修辑坊]
```

- 蓄鱼仓位于江州鱼市的**西侧**（蓄鱼仓 E→江州鱼市）
- 但江州鱼市的西出口指向**栖浪酒家**，而不是蓄鱼仓
- 这意味着两个不同的房间（蓄鱼仓、栖浪酒家）都占据了江州鱼市西侧的位置，**地图坐标冲突**

**修改建议：** 将蓄鱼仓的东出口改为连接渔户营（而非江州鱼市），符合实际地理位置（蓄鱼仓在清鳞分拣厂南方、渔户营西南方）：
```
蓄鱼仓 东 → 渔户营（从仓东侧小路返回渔户营）
渔户营 西南 → 蓄鱼仓
```

### 4.2 概览图结构与实际出口不符

概览图显示的线性关系：
```
渔户营 ↔ 江州鱼市 ↔ 海港商会 ↔ 栖浪酒家
```

但实际上 `海港商会` 和 `栖浪酒家` **互相之间没有连接**，它们都是分别连接江州鱼市。正确的表达应该是：
```
栖浪酒家 ↔ 江州鱼市 ↔ 海港商会
               ↑
            渔户营
```

### 4.3 云栖城的垂直坐标与基本方位混用

云栖城是悬崖城，本身具有强烈的垂直空间感，但设计中大量用「东西南北」描述本质是「高低上下」的关系（如 `青蔬坊 南→云舟修葺台（上行石阶）`），造成认知混乱。

**修改建议：** 为云栖城引入 `上(Up)/下(Down)` 出口类型（MUD标准支持），将垂直移动与水平移动区分开：

| 原设计 | 修改后 |
|--------|--------|
| 青蔬坊 南→云舟修葺台（上行） | 青蔬坊 **Up**→云舟修葺台 |
| 烟舟坊 东→青蔬坊（上行栈道） | 烟舟坊 **Up**→青蔬坊 或 保留东向但删除"上行"说明 |
| 烟舟坊 Up→云舟修葺台（竹篮） | 同时确保云舟修葺台 **Down**→烟舟坊 ✓ |

## 五、设计完整性问题（Minor）

### 5.1 地图设计灵感文档中「望霄楼」描述为空

```markdown
- 望霄楼：
```
灵感文档中望霄楼条目为空白，这说明设计师在灵感阶段对这个区域尚未形成清晰画面，后续在完整文档中补充了细节，但两文档未同步。

**修改建议：** 在灵感文档中补充：`望霄楼：倚崖而建的多层茶楼，登顶可俯瞰云海与通玄门，是云栖城视野最佳的观景地。`

### 5.2 NPC分布不均衡

| 区域 | NPC数量 |
|------|--------|
| 渔户营、栖浪酒家 | 3个 |
| 玄章镇潮阁 | 0个（守阁人无）+ 1个隐藏 |
| 海港商会 | 3个（含外来NPC） |

玄章镇潮阁作为主线高潮场景，缺乏常驻NPC，纯靠环境叙事（石碑、石砖）推进，互动感偏弱。

**修改建议：** 在阁一层增加一个「神龛」互动点或者一名「自发前来祭拜的老渔民」，既丰富现场感，也可为初到玩家提供导引性对话。

### 5.3 支线奖励明确程度不一致

| 支线 | 奖励明确度 |
|------|----------|
| 刮鳞小刀 → 修辑坊升级 | ✅ 清晰 |
| 护身符 → 通玄门护体 | ✅ 清晰 |
| 鱼丸秘方（吴婶→栖浪酒家） | ❌ 无奖励说明 |
| 云雾茶（青儿→望霄楼） | ❌ 奖励断链 |
| 悬索网检查（老金） | ❌ 无奖励说明 |

## 六、修改优先级总结

| 优先级 | 问题 | 影响 |
|--------|------|------|
| 🔴 P0 | 捕鱼房缺南出口；蓄鱼仓东出口无对应入口 | 玩家被困/导航断裂 |
| 🔴 P0 | 尾橹任务物品来源空缺 | 主线任务无法完成 |
| 🔴 P0 | 云栖城：烟舟坊↔市集南北互指、望霄楼↔市集方向矛盾 | 导航崩溃 |
| 🟠 P1 | 失踪渔船3人下落不明 | 剧情悬案不收尾 |
| 🟠 P1 | 洪大爷主线信息被付钱行为锁住 | 主线可能无法推进 |
| 🟠 P1 | 云雾茶断链（购买→用途不存在） | 任务道具无效 |
| 🟠 P1 | 玄章镇潮阁内部单向通行 | 无法返回 |
| 🟡 P2 | 修辑坊↔栖浪酒家方向矛盾（南≠西） | 方位混乱 |
| 🟡 P2 | 云栖城垂直移动用南北方向描述 | 认知混乱 |
| 🟢 P3 | 海港商会出口不完整 | 体验不完整 |
| 🟢 P3 | NPC分布不均（玄章镇潮阁空场） | 沉浸感弱 |
| 🟢 P3 | 部分支线无奖励说明 | 设计文档不完整 |
